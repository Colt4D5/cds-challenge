Challenge 2

Provided:

$(HandicapBtn").on("click", function(evt) {
  if ($(this).prop("checked")) {
    $("[id^=Row_HandicapText]").show();
    $("#HandicapText").addClass("required");
  } else {
    $("[id^=Row_HandicapText]").hide;
    $("#HandicapText").val("").removeClass("required");
  }
});

Solution:

$("#HandicapBtn").on("click", function(evt) {
  if ($(this).prop("checked")) {
    $("[id^=Row_HandicapText]").show();
    $("#HandicapText").addClass("required");
  } else {
    $("[id^=Row_HandicapText]").hide();
    $("#HandicapText").val("").removeClass("required");
  }
});

Edits:

  • Added open quotes on the initial element query on which the click event is attached.
  • Added the id selector* to the initial element query on which the click event is attached.
  • The .hide method is referenced on the "Row_HandicapText" queried element but is not invoked, so I added parenthesis to do just that.

Comments:

  • *I'm doing a bit of assuming about what the structure of the HTML looks like, but in order to select an element the initial element on which the click event is attached must be prefixed with either an id or class selector (depending on the HTML element).

Challenge 3

Provided:

SELECT
      [name] AS valuefield
      ,[longname] AS labelfield
      ,state_id
      regionId
    FROM TAB_SS_StateDefinitions
    WHERE(@Country = ''
      OR Country = @Country)
      AND Name <> XX
    ORDER BY
      labelfield

Solution:

SELECT
      [name] AS valuefield
      ,[longname] AS labelfield
      ,state_id
      regionId
FROM
    TAB_SS_StateDefinitions
    WHERE
      (@Country = '' OR Country = @Country)
      AND Nname <> XX
    ORDER BY
      labelfield;

Edits:

  • Name changed to lowercase to match SELECT query name.
  • Added semi-colon at end of query statement.
  • Format changes*

Comments:

  • I'm not too good with SQL, so that's about it as far as I can tell.
  • *I've also reformatted the styling a bit to make it a bit more readable, but I don't think it'll have any effect on its functionality...

Challenge 4

Provided:

Write a select statement from the TAB_Reg table that selects firstname, lastname, optin and email where optin equals 1 and order the results alphabetically:

Solution:

SELECT
      firstname
      ,lastname
      ,optin
      email
FROM
    TAB_Reg
    WHERE
      optin = 1
    ORDER BY
      lastname, firstname;

Comments:

  • Although not specified, I am assuming here that you are looking for the results to be alphabetically ordered by last name.
